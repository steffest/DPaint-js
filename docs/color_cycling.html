Color Cycling

Color cycling is a classic technique to animate a classic image by shifting colors within the palette range.
DPaint.js fully supports this. (See this example.)

How to set up a Color Cycle

  1. Open the Palette Panel.
  2. Click on the Ranges tab (next to the "Color" tab).
  3. Click the Add Range button. A new range control will appear.
  4. Define the Range:
    • Click on the color strip (preview bar) of the new range. The cursor will change, asking for the "From" color.
    • Click a color slot in your palette to set the Start of the range.
    • Click another color slot to set the End of the range.
  5. Use the active checkbox to enable/disable the cycle.
  6. Use the slider to adjust the speed and direction of the cycle.

Important Drawback: Avoid Identical Colors

Because the engine maps pixels to palette indices, it needs to know which index corresponds to which color. If your palette contains duplicate colors (e.g., two slots both defining pure black #000000), the engine gets confused about which index to use.

This can cause issues with Color Cycling or pixel-perfect index manipulation, as the engine might accidentally map your pixels to the wrong index (e.g., Index 5 instead of Index 10, because they are identical).

Tip:When using color-cycling, always try to keep your palette colors unique.

(Yes, I know this is a bit of a bummer, and very different from how traditional indexed color engines worked, but it's a trade-off I had to make to achieve the hybrid true-color/indexed-color approach of DPaint.js.)
